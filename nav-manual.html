05 · 导航设计规则
R1
最高频操作最显眼：「新对话」用橙色主色按钮，其余导航项用文字+图标列表形式，视觉权重清晰区分。
R2
层级最多两层：侧边栏只有「分组标题 → 子项」，不做三级菜单。参照 Claude.ai 的扁平结构。
R3
未上线产品可见不可用：菠萝聊 / 菠萝卡显示在列表，透明度 50%，角标「即将上线」，不可点击。制造期待，不欺骗用户。
R4
C 端优先，B 端收底：便利店赋能（店主入驻）放在侧边栏最下方。主动线是 C 端用户问答，不混淆受众。
R5
合并冗余项：原「软件 / 文档 / 商店 / 关于我们 / 联系我们」5 项合并为「C 家族 / 关于 Nanasity」2 项，减少认知负荷。
R6
配色守则：主色 #c96442（暖橙），背景 #f5f4f0，侧边栏白底 #fff。禁止暗色主题，保持暖色极简白底风格。
